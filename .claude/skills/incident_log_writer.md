# incident_log_writer
Trigger: recovery, restoration, rollback, file loss, unexpected behavior, root cause found.

When incident occurs/resolved, append to `docs/INCIDENTS.md`:
YYYY-MM-DD HH:MM — [short title]
What happened: [1-2 sentences]
Root cause: [actual cause, not symptom]
Fix: [what was done]
Prevention: [rule/check added to avoid repeat]
Related TZ: [TZ-XXX]

If `docs/INCIDENTS.md` doesn't exist — create it with header.

Do not skip this step on "small" incidents — patterns emerge from accumulation.
