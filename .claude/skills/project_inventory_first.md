# Skill: project_inventory_first

## Trigger

Apply when ANY of these match:
- New chat session AND task involves adding/changing functionality
- Task involves creating a new module, feature, service, or package
- Task phrase includes: new module, add feature, implement X, write Y, create Z
- Before нарезать TZ (issuing ТЗ to Code) for any non-trivial implementation

## Rule: 4 Mandatory Inventory Checks

Perform ALL four checks before designing the TZ. Do not skip any.

### Check 1 — Walk active tree (top-2 levels)
```
tree src/ services/ scripts/ handlers/ telegram_ui/ collectors/ /F /L 2
```
OR equivalent glob scan. Goal: know what packages/modules actually exist today.

### Check 2 — Grep for keyword in active + restored
```
grep -r "feature_keyword" src/ services/ _recovery/restored/ --include="*.py" -l
```
Substitute `feature_keyword` with the concept being implemented. Goal: detect parallel implementations.

### Check 3 — Read PROJECT_MAP.md if it exists
```
docs/STATE/PROJECT_MAP.md  (generated by state_snapshot.py)
```
Check the "active modules" and "conflicts" sections. If file does not exist, skip and note it.

### Check 4 — Read RESTORED_FEATURES_AUDIT if available
```
docs/STATE/RESTORED_FEATURES_AUDIT_*.json  (latest)
```
Check `modules[].status`, `modules[].recommendation`, `modules[].conflict` fields. If `recommendation == "leave_as_restored"` for any module relevant to the TZ — STOP and revise scope.

## Forbidden

- Нарезать TZ without completing these checks.
- Rely solely on Anthropic training knowledge ("I know this module exists...").
- Assume a module is absent without checking `_recovery/restored/`.
- Assume a module is present without grep confirmation.

## Recovery: If inventory reveals existing code

Replace the planned TZ with: `TZ-INVENTORY-<feature>` that:
1. Documents what was found (path, status, conflicts)
2. Decides: reactivate_as_is / reactivate_plus_integrate / leave_as_restored
3. Issues targeted integration TZ — NOT a full reimplementation

## Why this skill exists

**INC-012** (2026-04-29): Architect designed TZ-PROJECT-MEMORY-DEFENSE *after* discovering (mid-session) that:
- `services/advise_v2/cascade.py` was a parallel re-implementation of `_recovery/restored/src/advisor/v2/cascade.py`
- `src/features/calendar.py` was missing from active code, causing 2 broken test files
- 34 modules existed in `_recovery/restored/` with no active counterpart
- 1098 pre-computed feature parquets were available but unknown to the architect

All of these would have been visible with a 5-minute grep pass before writing the first TZ.
