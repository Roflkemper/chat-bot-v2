V12.2 PRE-EDGE

Что добавлено:
- новый PRE-EDGE alert, когда цена подошла к краю диапазона, но edge ещё не подтверждён
- alert не даёт вход, а только предупреждает: готовиться и ждать confirm
- антиспам сохранён: одинаковые PRE-EDGE состояния не дублируются до cooldown
- логика использует диапазон, локацию цены и readiness / confirm_ready

Когда должен писать:
- UPPER_EDGE -> контекст SHORT
- LOWER_EDGE -> контекст LONG
- edge = NONE / WEAK
- confirm_ready = 0
- total readiness >= 40

Когда не должен писать:
- середина диапазона
- готовый actionable edge уже есть (тогда идёт обычный EDGE ALERT)
- состояние не изменилось и cooldown ещё не истёк
