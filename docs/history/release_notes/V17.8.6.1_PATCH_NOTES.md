# V17.8.6.1 — historical pattern store + regime-aware matching + decision integration

## Что добавлено
- `storage/pattern_history_store.py` — файловое хранилище исторических паттернов (`jsonl`)
- `features/regime_aware_pattern_engine.py` — regime-aware matching, фильтры значимости, top-k matches
- `core/forecast_decision_adapter.py` — интеграция forecast consensus в execution / decision confidence
- `tests/test_regime_aware_pattern_engine.py` — минимальные smoke tests

## Логика пакета
1. Паттерны сохраняются в историю с полями:
   - tf
   - market_regime
   - range_position
   - direction
   - future_move_pct
   - horizon_bars
   - normalized closes
2. Поиск совпадений идет только по совместимому контексту:
   - тот же TF
   - совместимый market_regime
   - совместимый range_position
3. Слабые паттерны отфильтровываются:
   - `MIN_MOVE_THRESHOLD`
   - `MIN_SAMPLE_COUNT`
   - `WIN_RATE_THRESHOLD`
4. Forecast consensus передается в decision layer как adjustment:
   - `HIGH` alignment усиливает подтвержденный сценарий
   - `CONFLICT` режет confidence и запрещает агрессивный execution
   - `1d` veto ослабляет контртрендовые входы

## Встраивание
- history store можно подключить в pipeline после формирования закрытого окна
- decision adapter вызывается после `forecast_consensus`, до финального render/action

## Что пакет не делает
- не добавляет ML
- не добавляет автоторговлю
- не пишет в биржу
