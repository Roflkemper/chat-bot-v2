# V17.8.6.2.3 — HARD CUTOVER RELEASE

## Что делает пакет
- вводит единый `execution_snapshot` как source of truth
- режет legacy fallback для state/consensus/pattern
- форсит `SEARCH_TRIGGER`, если цена уже внутри активного блока
- скрывает слабый pattern-memory ниже порога `0.20%`
- считает `block_depth_pct`
- считает `distance_to_active_edge`, `distance_to_upper_edge`, `distance_to_lower_edge`
- переводит Telegram-рендер на новый snapshot

## Что внутри
- `core/execution_snapshot.py`
- `core/hard_cutover_release.py`
- `renderers/hard_cutover_renderer.py`
- `tests/test_hard_cutover.py`

## Ограничение
Это overlay-пакет. Для drop-in интеграции нужно подключить `apply_hard_cutover(...)` в боевой pipeline и отключить legacy formatter.
