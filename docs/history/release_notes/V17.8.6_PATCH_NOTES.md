# V17.8.6 — MULTI-TF FORECAST LAYER

## Что внутри
- `services/timeframe_aggregator.py` — агрегация закрытых 1h свечей в 4h и 1d
- `features/multi_tf_forecast.py` — базовые фичи forecast-слоя:
  - normalizer
  - similarity_score
  - short-term momentum/pressure proxy
  - session context
  - daily phase detection
- `core/forecast_consensus.py` — арбитр 1h / 4h / 1d
- `renderers/forecast_block.py` — компактный Telegram-блок прогноза

## Что это за пакет
Это overlay-пакет под V17.8.x. Он не знает сигнатуры всех текущих сервисов репозитория, поэтому дан как безопасный модульный слой, который можно встроить в существующий pipeline:

`data -> features -> context -> decision -> execution -> render`

## Что нужно подключить вручную в проекте
1. В data layer передавать в forecast-модуль массив закрытых 1h свечей.
2. Собрать из них `4h` и `1d` через `aggregate_candles`.
3. В current context сформировать:
   - short-term features
   - session context
   - medium context
4. В decision layer вызвать `build_consensus(...)`.
5. В renderer подключить `render_forecast_block(...)`.

## Что ещё не включено в этот overlay
- поиск исторических pattern matches по базе 2024/2025/2026
- regime/range-position persistence storage
- подключение к state machine confidence
- жёсткий veto на aggressive execution
