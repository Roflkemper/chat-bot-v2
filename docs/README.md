# docs

`docs/` хранит исторические и служебные материалы, которые не должны засорять корень проекта.

## Разделы
- `history/release_notes/` — старые release notes, patch notes, hotfix notes
- `history/checklists/` — старые launch checklists
- `history/manifests/` — старые release manifests
- `history/misc/` — вспомогательные заметки и архивные инструкции
