## 3a. Root cause analysis — WHY RED

**The RED verdict is primarily a stale-init artifact, NOT evidence of engine logic bugs.**

### Stale-init trigger cascade (main failure mode)

All 6 simulated bots were initialized from the first available snapshot (2026-04-28T08:10).
No pre-window snapshots exist (tracker started April 28 only).

The init algorithm reconstructs N synthetic orders spread around `avg_entry` using the
grid step formula. For SHORT bots with `avg_entry ≈ 75,155` and current price `≈ 74,800`:

- Synthetic orders are spread from ~73,700 (low) to ~76,600 (high)
- `trigger_price = entry × (1 - target/100) = entry × 0.9975`
- Orders with `entry > 74,988` have `trigger > 74,800` → **trigger is already hit**
- Estimated: **~60–70 out of 128 synthetic orders trigger immediately** on bar 1

Effect:
1. Massive artificial Out Stop group fires within first few bars
2. sim.worst_upnl goes deeply negative (real: always positive +46 to +66 USDT)
3. After flush: `last_in_price` is set too high (~76,500) → no new INs open (BTC never reaches that level on Apr 28)
4. sim.volume and sim.delta_pnl collapse to a fraction of real values

**For LONG bots** (opposite effect): synthetic LONG orders trigger quickly as price rises above avg,
producing sim volume/PnL 2-3× higher than real.

### Why engine logic is likely OK

The existing W1/W2/W3 validation (replay_window.py) passed for TEST_1/2/3 at windows on
2026-04-17 and 2026-04-20 using proper warmup init (fresh snapshot before window).
The engine code matches GINAREA_MECHANICS documentation (§3 cross-check above).

The reconcile failure is about **initialization quality**, not engine correctness.

### Required fix before re-run

**TZ-ENGINE-FIX-STALE-INIT**: Run simulation from each bot's start date (~April 16 for TEST_1/3,
~April 24 for TEST_2) with zero initial position, through to April 28. Compare April 28 state
to real snapshot. This eliminates the trigger cascade and produces meaningful comparison.
Risk: operator interventions during April 16-28 are not captured → expect some drift.
