# Глоссарий: технические коды → человеческие имена

Чтобы в TG-сообщениях, отчётах и обсуждениях все разговаривали на одном
языке, фиксируем русские названия для технических кодов стратегий и сценариев.

---

## Стратегии grid-ботов

| Технический код | Русское имя | Описание |
|---|---|---|
| `baseline` | **«Без вмешательства»** | Сетка работает как сейчас — никто не пауcтит, не выгружает, границы не двигает. Эталон для сравнения. |
| `pause_on_drawdown` | **«Защита от слива»** | Пауза новых ордеров при unrealized < -$30 + возврат после отката >50%. Жертвует объёмом ради защиты. |
| `partial_unload_on_retrace` | **«Частичная фиксация»** | Если был пик прибыли > $5 и она откатилась >40% — выгружаем 30% позиции. |
| `trend_chase` | **«Подъём границ за трендом»** | При движении цены >0.5% за час — поднимаем верхнюю границу сетки на 2%. Не даёт боту замёрзнуть за boundaries. |
| `combined` | **«Защита+Фиксация+Догон»** (комбинированная) | Все три правила вместе — лучший компромисс между объёмом и защитой. |

**В TG-сообщениях** используем русские имена. В коде — английские константы.

## Применение в коде

**Не переименовываем** код-уровневые константы (это сломает API и тесты).
**Переименовываем** только то что видит оператор:
- TG-сообщения от сервисов
- Markdown отчёты в `docs/`
- Названия в TG-командах (`/bots_kpi`, `/audit`)

Для каждого технического кода в TG-форматтерах вызываем функцию-переводчик
`humanize_setup_type()` / `humanize_scenario()` (которые возвращают русское имя).
