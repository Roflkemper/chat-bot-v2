# PROJECT MANIFEST


## DEVELOPMENT PROTOCOL (MANDATORY)

- Отдавать пользователю только готовые файлы для замены в проекте.
- Не добавлять воду и лишние описания, если пользователь сам не просит объяснение.
- Не присылать куски кода в чат.
- Перед выдачей решения максимально проверять результат, чтобы это не были косметические заглушки.
- Убеждаться, что слои и функции реально работают и получают данные из источников (например, Binance), если это предусмотрено логикой.
- Работать строго по согласованному плану.


## RUNTIME DELIVERY RULES
- Не присылать код в чат.
- Не просить пользователя вручную вставлять строки в файлы.
- Отдавать готовые файлы или полный ZIP-релиз.
- Перед выдачей проверять, что изменения рабочие и не являются косметикой или заглушкой.
- Для GRID-логики при нейтральном bias в середине диапазона приоритет стороны = NEUTRAL.
- GRID-импульс считать от текущей цены до ближайшей релевантной ликвидности, а не от произвольного диапазонного ориентира.
- Показывать активацию слоёв сеток явно по порогам 1.3% / 2.2% / 2.9%.
- Разделять выводы TRADER VIEW и GRID VIEW.
