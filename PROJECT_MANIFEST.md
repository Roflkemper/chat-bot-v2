# PROJECT MANIFEST


## DEVELOPMENT PROTOCOL (MANDATORY)

- Отдавать пользователю только готовые файлы для замены в проекте.
- Не добавлять воду и лишние описания, если пользователь сам не просит объяснение.
- Не присылать куски кода в чат.
- Перед выдачей решения максимально проверять результат, чтобы это не были косметические заглушки.
- Убеждаться, что слои и функции реально работают и получают данные из источников (например, Binance), если это предусмотрено логикой.
- Работать строго по согласованному плану.
- Использовать PROJECT_MANIFEST.md как обязательную основу для всех следующих сборок.
- Не присылать код в чат. Отдавать только готовые файлы или полный ZIP-релиз.
- Не писать пользователю, что ему нужно вставить строку в файл вручную.
- Не отдавать косметические фиксы и заглушки; отдавать только рабочие изменения.
- Перед выдачей проверять реальную работоспособность изменённых модулей и критические сценарии.
