# V17.8.6.2 — PATTERN STABILITY + EXECUTION SYNC

## Что добавлено
- `core/pattern_stability.py` — фильтр стабильности для pattern-memory:
  - anti-flip logic
  - min persistence bars
  - direction lock при слабом новом сигнале
  - ослабление до `NEUTRAL/LOW`, а не мгновенный переворот
- `core/execution_zone_sync.py` — синхронизация зоны, pattern и execution:
  - `SEARCH_TRIGGER` если цена внутри активного блока
  - `depth inside block`
  - `distance to edge`
  - `OVERRUN` при глубоком проникновении
  - consensus normalization: не давать пустой `NEUTRAL`, если есть рабочая сторона
- `renderers/compact_execution_sync.py` — компактный Telegram-блок:
  - статус
  - действие
  - зона
  - глубина
  - расстояние до края
  - consensus
  - trigger / invalidation / hedge arm
- `tests/test_pattern_stability.py` — smoke tests по anti-flip и zone sync

## Логика пакета
1. **Зона первична.**
   - если цена в активном блоке, state должен быть `SEARCH_TRIGGER`, а не `PAUSE`
2. **Паттерн не имеет права резко переворачиваться без persistence.**
   - новый противоположный weak/mid сигнал сначала режется stability layer
3. **Consensus обязан быть синхронным с execution.**
   - если есть рабочая сторона, вывод: `SHORT | LOW`, `LONG | MID`, либо `CONFLICTED`
4. **Depth влияет на решение.**
   - >50%: понижение confidence / aggressiveness
   - >85%: `OVERRUN`

## Что пакет не делает
- не добавляет ML
- не добавляет автоторговлю
- не пишет в биржу
- не меняет сигнатуры существующих сервисов насильно

## Как подключать
1. После расчета raw pattern вызвать `stabilize_pattern_signal(...)`
2. После расчета range/block context вызвать `build_execution_zone_state(...)`
3. В текущем execution/advisor заменить текстовый `PAUSE у края` на новый state
4. В renderer подключить `render_compact_execution_sync(...)`
