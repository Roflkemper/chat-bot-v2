# V17.8.6.2.2 — PIPELINE INTEGRATION FIX

## Что чинит
- Подключает state/depth/distance поля в итоговый decision snapshot.
- Запрещает `MID_RANGE` / `PAUSE`, если цена уже внутри активного блока.
- Нормализует consensus: если execution side уже есть, `NEUTRAL` не допускается.
- Скрывает слабые pattern-memory сигналы ниже порога значимости.
- Даёт компактный renderer, который реально использует новые поля.

## Файлы
- `core/pipeline_integration_fix.py`
- `renderers/pipeline_execution_renderer.py`
- `tests/test_pipeline_integration_fix.py`

## Как встраивать
1. После расчёта state-machine вызвать `build_pipeline_fields(...)`.
2. Результат смержить в общий `decision_snapshot` через `merge_into_decision_snapshot(...)`.
3. В Telegram использовать `render_compact_execution(snapshot, ...)` либо прокинуть новые поля в существующий formatter.
4. Старые поля `QUIET/PAUSE` и `consensus: NEUTRAL` больше не брать как источник истины.

## Ключевые правила
- Зона первична.
- State machine первичнее старого runtime label.
- Pattern ниже порога не показывать.
- Consensus не должен спорить с execution side.
